### По заданным описаниям таблиц и вставки данных определить используемый движок
---
```sql
CREATE TABLE tbl1
(
    UserID UInt64,
    PageViews UInt8,
    Duration UInt8,
    Sign Int8,
    Version UInt8
)
ENGINE = <ENGINE>
ORDER BY UserID;
```
Здесь используется движок `CollapsingMergeTree(Sign)`, который удаляет строки с противоположными значениями в указанном поле.

---
```sql
CREATE TABLE tbl2
(
    key UInt32,
    value UInt32
)
ENGINE = <ENGINE>
ORDER BY key;
```
Судя по сумме значений поля `value` у строки с ключом 1, движок должен быть `SummingMergeTree`

---
```sql
CREATE TABLE tbl3
(
    `id` Int32,
    `status` String,
    `price` String,
    `comment` String
)
ENGINE = <ENGINE>
PRIMARY KEY (id)
ORDER BY (id, status);
```
Здесь `ReplacingMergeTree`, потому что осталась одна последняяя строка

---
```sql
CREATE TABLE tbl4
(   CounterID UInt8,
    StartDate Date,
    UserID UInt64
) ENGINE = <ENGINE>
PARTITION BY toYYYYMM(StartDate)
ORDER BY (CounterID, StartDate);
```
Движок `MergeTree`


```sql
CREATE TABLE tbl5
(   CounterID UInt8,
    StartDate Date,
    UserID AggregateFunction(uniq, UInt64)
) ENGINE = <ENGINE>
PARTITION BY toYYYYMM(StartDate)
ORDER BY (CounterID, StartDate);
```
Тут должен быть движок `AggregatingMergeTree`, потому что этот движок хранит состояния, 

---
```sql
CREATE TABLE tbl6
(
    `id` Int32,
    `status` String,
    `price` String,
    `comment` String,
    `sign` Int8
)
ENGINE = <ENGINE>
PRIMARY KEY (id)
ORDER BY (id, status);
```
Здесь используется `CollapsingMergeTree(sign)`, потому что удалились строки с противоположным значением ключа `sign`
